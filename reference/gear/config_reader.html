Workflow Configuration file reader¶

Implementation of a TOML-like configuration file for HADDOCK3.

HADDOCK3 user configuration files follow TOML syntax plus additional features required for HADDOCK3. Therefore, we have implemented our own TOML-like parser to accommodate the extra features needed for HADDOCK3.

The most relevant features of HADDOCK3 user configuration files are:

Accepts repeated keys: blocks can have repeated names, for example:

[topoaa]
# parameters here...

[rigidbody]
# parameters here...

[caprieval]
# parameters here...

[flexref]
# parameters here...

[caprieval]
# parameters here...

Allows in-line comments:

[module]
parameter = value # some comment

The following types are implemented:

  • strings

  • numbers

  • null/none

  • nan

  • lists defined in one lines

  • lists defined in multiple lines (require empty line after the list definition)

  • datetime.fromisoformat

To efficiently use this module, see functions:

  • haddock.gear.config_reader.read_config()

  • haddock.gear.config_reader.get_module_name()

Parameters:
